== Keystore lokal erstellen

Öffne ein Terminal (in IntelliJ unten links) und nutze das keytool (ist Teil deines JDKs). Ersetze die Platzhalter in Klammern:

[source]
----
keytool -genkey -v -keystore shipermansfriend_release.keystore -alias shipermansfriend-alias -keyalg RSA -keysize 2048 -validity 10000
----

Wichtig: Merk dir das Passwort gut!
Die Datei release.keystore wird in dem Ordner erstellt, in dem du gerade bist.

== Keystore für GitHub Actions vorbereiten

Damit GitHub dein APK signieren kann, ohne dass du das Passwort in den Code schreibst:

Wandle den Keystore in einen Text-String um (Base64), damit du ihn als "Secret" speichern kannst:

[source]
----
[Convert]::ToBase64String([IO.File]::ReadAllBytes("E:\Programmierung\IntelliJ\keystore\shipermansfriend_release.keystore"))
----

- Kopiere diesen langen String.Gehe in dein GitHub-Repo: Settings -> Secrets and variables -> Actions.
- Erstelle ein neues Secret: RELEASE_KEYSTORE_BASE64 (füge den String ein).
- Erstelle weitere Secrets für: RELEASE_KEYSTORE_PASSWORD, RELEASE_KEY_ALIAS und RELEASE_KEY_PASSWORD.

=== RELEASE_KEYSTORE_PASSWORD

(Das Haustür-Passwort)
Das ist das Passwort für die gesamte Keystore-Datei (release.keystore). Ohne dieses Passwort kann Gradle die Datei nicht einmal öffnen.Was kommt rein? Das Passwort, das du am Anfang des keytool-Befehls eingegeben hast.

=== RELEASE_KEY_ALIAS

(Der Name des Schlüssels)
Ein Keystore kann mehrere Schlüssel enthalten (wie ein Schlüsselbund). Der Alias ist der Name des spezifischen Schlüssels für deine App.Was kommt rein? In meinem Beispiel-Befehl war das news-home-alias. Wenn du den Befehl so kopiert hast, ist das dein Alias.

=== RELEASE_KEY_PASSWORD

(Das Zimmertür-Passwort)
Das ist das Passwort für den spezifischen Schlüssel innerhalb des Keystores.Hinweis: Oft setzt man dieses Passwort beim Erstellen identisch mit dem Keystore-Passwort (viele Entwickler machen das, um es einfacher zu haben).Was kommt rein? Das Passwort, das du ganz am Ende des keytool-Prozesses vergeben hast (oder einfach bestätigt hast, um dasselbe wie für den Keystore zu nutzen).

== Gradle konfigurieren

Passe den android-Block an, damit er die Secrets nutzt (lokal nutzt er Fallbacks oder du trägst sie in deine lokale local.properties ein):

[gradle]
----
android {
// ... namespace, compileSdk etc.

    signingConfigs {
        create("release") {
            // Wir lesen die Daten aus Umgebungsvariablen (für GitHub)
            // oder aus Projekteigenschaften (lokal)
            storeFile = file(System.getenv("RELEASE_KEYSTORE_PATH") ?: "release.keystore")
            storePassword = System.getenv("RELEASE_KEYSTORE_PASSWORD") ?: project.findProperty("RELEASE_STORE_PASSWORD").toString()
            keyAlias = System.getenv("RELEASE_KEY_ALIAS") ?: project.findProperty("RELEASE_KEY_ALIAS").toString()
            keyPassword = System.getenv("RELEASE_KEY_PASSWORD") ?: project.findProperty("RELEASE_KEY_PASSWORD").toString()
        }
    }

    buildTypes {
        getByName("release") {
            isMinifyEnabled = true // Aktiviert R8 Optimierung & Obfuscation
            signingConfig = signingConfigs.getByName("release")
            proguardFiles(getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro")
        }
    }
}
----

== GitHub Workflow anpassen

[github]
----
- name: Decode Keystore
env:
ENCODED_STRING: ${{ secrets.RELEASE_KEYSTORE_BASE64 }}
run: |
echo $ENCODED_STRING | base64 -d > shipermansfriend/composeApp/release.keystore

- name: Build Release APK
working-directory: shipermansfriend
env:
RELEASE_KEYSTORE_PASSWORD: ${{ secrets.RELEASE_KEYSTORE_PASSWORD }}
RELEASE_KEY_ALIAS: ${{ secrets.RELEASE_KEY_ALIAS }}
RELEASE_KEY_PASSWORD: ${{ secrets.RELEASE_KEY_PASSWORD }}
RELEASE_KEYSTORE_PATH: release.keystore
run: ./gradlew assembleRelease
----
